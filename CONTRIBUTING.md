# Принос към проекта

Тази книга е „жив" документ и приветстваме всякакъв вид принос — от поправка на правописна грешка до цяла нова глава.

---

## Какво търсим

- **Поправки в текста** — правопис, граматика, неточни формулировки
- **Технически корекции** — грешки в код, XML, SQL примери
- **Нови казуси** — реални сценарии от практиката със SAF-T
- **Подобрения на кода** — Rust, Python, SurrealQL, PostgreSQL
- **Допълнения към приложенията** — нови сметки, данъчни кодове, мерни единици
- **Превод** — секции на английски за международната аудитория

## Какво НЕ приемаме

- Споменаване на конкретни търговски марки на счетоводен софтуер
- Правни анализи и тълкувания на закони (книгата е техническа)
- Генериран текст без проверка от човек
- Съдържание, което нарушава лиценза MIT

## Как да допринесете

### 1. Отворете Issue

Ако сте намерили грешка или имате идея, но не искате да пишете код — отворете [Issue](../../issues). Опишете:

- Коя глава / секция засяга
- Какъв е проблемът или предложението
- Ако е грешка в код — какъв е очакваният резултат

### 2. Направете Pull Request

```bash
# Клонирайте репото
git clone https://github.com/katehonz/saf-t-bg.git
cd saf-t-bg

# Създайте нов клон
git checkout -b fix/chapter-3-typo

# Направете промените си
# ...

# Комитнете
git add .
git commit -m "Fix typo in Chapter 3 section 3.2"

# Изпратете
git push origin fix/chapter-3-typo
```

След това отворете Pull Request от GitHub.

### 3. Конвенции за именуване на клонове

| Тип | Формат | Пример |
|-----|--------|--------|
| Поправка | `fix/описание` | `fix/chapter-5-uom-mapping` |
| Нова функционалност | `feat/описание` | `feat/chapter-12-e-invoicing` |
| Документация | `docs/описание` | `docs/appendix-new-tax-codes` |

### 4. Конвенции за комит съобщения

Кратко, на английски, в imperative mood:

```
Fix incorrect TaxCode in Chapter 3 example
Add new case study: logistics company with 5 warehouses
Update UOM table with 2026 codes
```

## Структура на файловете

- Всяка глава е в отделен файл: `CHAPTER_X.md`
- Пълният текст е огледален в `FULL_BOOK.md` — ако редактирате глава, обновете и него
- Резюметата са в `SUMMARY.md` — ако добавите глава, добавете и резюме
- Планът е в `PLAN.md` — отразете промените и там

## Стил на писане

- **Тон**: Директен, технически, без излишна дипломатичност
- **Език**: Български, с технически термини на английски в скоби при първо споменаване
- **Код**: С коментари на български, обяснящи счетоводния смисъл
- **Примери**: Реалистични, с български контекст (ЕИК, ДДС, НАП)
